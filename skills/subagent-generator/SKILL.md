# subagent-generator

Выход: валидный `.claude/agents/<name>.md`. Канон и контракт живого скилла —
`../runewright-blueprint/SKILL.md` (прочитай перед работой). Иди строго по шагам.

**Скилл или сабагент?** Сабагент нужен, только если верно хотя бы одно:
- работа изолируется от контекста оркестратора (грязный поиск, длинный вывод инструментов);
- нужен свой набор инструментов/прав (например, read-only ревьюер без Write);
- задача запускается параллельно с другими (N независимых кусков работы).
Иначе — остановись и предложи skill-generator. Работаешь в Codex (нет `.claude/`,
есть `.agents/`) — сабагенты недоступны, предложи оформить роль как скилл.

## Шаги

1. **Прочитай уроки:** последние 5 записей `.runewright/feedback/subagent-generator.md`
   (нет файла — пропусти). Проверь дубликаты: фронтматтер существующих
   `.claude/agents/*.md` и агентов активных плагинов. Роль уже есть — предложи
   доработку и остановись.

2. **Собери входные данные.** Есть бриф `.runewright/briefs/<name>.md` с секцией
   «Исполнитель: скилл + сабагент» — возьми данные оттуда, переспроси недостающее.
   Иначе спроси одним сообщением:
   - роль в одном предложении и границы: что делает и что ЗАПРЕЩЕНО;
   - когда оркестратор его вызывает и что передаёт (агент НЕ видит историю диалога —
     вход должен быть самодостаточным);
   - какие скиллы/источники истины использует;
   - что возвращает и как оркестратор проверит сдачу;
   - минимальные инструменты: read-only роли — `Read, Glob, Grep`; пишущие —
     + `Write, Edit`; запускающие — + `Bash`. Не давай `*` без причины.

3. **Заполни шаблон** (ниже). Правила: `name` kebab-case = имя файла; description
   ≤ 500 символов по формуле «Роль. Вызывать когда... НЕ вызывать когда...» —
   оркестратор выбирает агента только по этому полю; тело ≤ 100 строк; знания —
   в источниках; финальное сообщение агента — отчёт фиксированного формата
   (оркестратор его парсит, а не читает).

4. **Создай файлы:** `.claude/agents/<name>.md` и лог
   `.runewright/feedback/agent-<name>.md`.

5. **Самопроверка** (все — да, иначе исправь):
   - [ ] description отвечает «когда вызывать» и «когда НЕ вызывать»
   - [ ] `tools` — минимальный список, не `*`
   - [ ] есть «Запрещено» — явные границы
   - [ ] «Вход» самодостаточен (без «см. выше по диалогу»)
   - [ ] формат отчёта фиксирован, содержит статус по критериям сдачи
   - [ ] секция «Параллельность»: N копий и как делить работу, либо почему один
   - [ ] тело ≤ 100 строк, YAML валиден

6. **Заверши:** покажи файл + пример вызова из оркестратора (1–2 строки).
   Допиши запись в свой feedback-лог.

## Шаблон .claude/agents/<name>.md

```markdown
---
name: <kebab-case-имя>
description: >-
  <Роль, 1 предложение.> Вызывать когда: <условия из плана оркестратора>.
  НЕ вызывать когда: <анти-триггеры — какая роль берёт это вместо>.
tools: <минимальный список через запятую>
---

Ты — <роль в одном предложении>.

## Вход
Оркестратор передаёт: <пути, ID задачи, критерии>. Чего-то нет во входе —
верни отчёт со статусом BLOCKED и списком недостающего, не додумывай.

## Источники истины
- `<ID/путь>` — <что брать>. Факта нет — статус BLOCKED, не выдумывай.

1. <действие> → <проверяемый результат>
N. Допиши строку исхода в `.runewright/feedback/agent-<имя>.md`.

- <файлы/действия вне зоны ответственности — по пункту на строку>

<«Можно N копий; делить по <файл/модуль/роут>; копии не пишут в одни файлы»

## Критерии сдачи
- [ ] <проверяемое условие>

## Формат отчёта (финальное сообщение)
СТАТУС: DONE | PARTIAL | BLOCKED
РЕЗУЛЬТАТ: <пути файлов или вывод>
ПРОВЕРЕНО: <какие критерии выполнены, чем подтверждено>
ПРОБЛЕМЫ: <что не удалось и почему | «нет»>
```

## Заметка для оркестратора (включай в скиллы, которые зовут агентов)
- Независимые куски → запускай агентов одним сообщением параллельно.
- PARTIAL/BLOCKED — не пересказывай: дошли вход, перезапусти или эскалируй.
- Не передавай агенту работу, которую он должен «додумать» из истории диалога.

## Критерии результата
Сдано: агент создан, 7 пунктов самопроверки пройдены, лог создан.
Хорошо: агент, чей отчёт оркестратор обрабатывает без чтения тела работы.
Плохо: агент с tools `*` и входом «делай как обсуждали».
